# コンソールRPG 実装課題 ― フェーズ5：コレクションで複数のモンスターと連戦する

## このお題のゴール

いまのバトルは、勇者 vs モンスター1体で終わりです。
このフェーズ5では、**モンスターを複数用意して、順番に連戦** させます。スライムを倒したら次はゴブリン、それも倒したら最後はドラゴン。全部倒せば勇者の勝ち、途中で力尽きればゲームオーバーです。

このフェーズ5で身につく内容：

- **コレクション**（`List` … 同じ種類のものをまとめて持つ入れ物）
- **`ArrayList`**（`List` の実物。要素を追加できる）
- **ジェネリクス**（`List<Monster>` … 「モンスターだけが入るリスト」と型で示す）
- **拡張for文**（`for (Monster m : monsters)` … リストの中身を1件ずつ取り出す）
- **ループの入れ子**（「敵ごとのループ」の中に「殴り合いのループ」が入る）

> `Character` / `Hero` / `Monster` は **今回も変更しません**。直すのは `Battle` と `Main` だけです。
> フェーズ4で進行を `Battle` に切り出しておいたので、**戦いのルール変更が `Battle` の中だけで完結する**ことを体感してください。

## 予備知識：`List` の使い方

今回はじめて出てくる書き方です。まずここだけ押さえてください。

```java
import java.util.ArrayList;   // ← ファイルの先頭に必要
import java.util.List;

List<Monster> monsters = new ArrayList<>();   // 空のリストを作る
monsters.add(new Monster("スライム", 15, 4));  // 末尾に追加する
monsters.add(new Monster("ゴブリン", 25, 6));

for (Monster m : monsters) {   // 中身を1件ずつ取り出す（拡張for文）
    m.showStatus();
}
```

- `List<Monster>` の `<Monster>` は「**この箱にはモンスターしか入れません**」という宣言です。おかげで、取り出すときに型を書き直す必要がありません。
- `List` と `ArrayList` が両方出てくるのが最初は不思議ですが、いまは **「`List` と書いて `new ArrayList<>()` する」がお決まりの形** とだけ覚えて先に進んでOKです（理由は後のフェーズで扱います）。
- `import` を書き忘れるとコンパイルエラーになります。エラーが出たら、まず先頭の `import` を疑ってください。

## クラス仕様：`Battle`（改造する）

### フィールド

| フィールド名 | 型 | 修飾子 | 意味 |
|---|---|---|---|
| `hero` | `Hero` | `private` | 勇者（変更なし） |
| `monsters` | `List<Monster>` | `private` | **これから順番に戦う敵の一覧**（`monster` 1体から変更） |

### コンストラクタ

```java
public Battle(Hero hero, List<Monster> monsters)
```

- モンスター1体ではなく、**リストごと** 受け取るように変える。

### メソッド

#### `public void start()`
- **やること**：敵のリストを頭から順に取り出し、1体ずつ戦わせる。
  1. 勇者の状態を表示する
  2. リストの中身を **拡張for文** で1体ずつ取り出す
     1. **勇者がすでに倒れていたら、そこでループを抜ける**（次の敵とは戦わせない）
     2. その1体との戦いを行う（`fight(...)` を呼ぶ）
  3. 全部終わったら結果を表示する（`showResult()`）
- **ヒント**：`start()` はもう殴り合いの中身を持ちません。**「敵を順番に呼び出す係」** に徹します。

#### `private void fight(Monster monster)`  ← 今回の新メソッド
- **やること**：**モンスター1体との戦い**。中身は、フェーズ4の `start()` に書いてあった `while` ループがほぼそのまま入ります。
  1. 遭遇メッセージを表示する（例：`勇者 は スライム に遭遇した！`）
  2. `while` ループで、両者が生きている間 殴り合う
     1. 勇者がモンスターを攻撃する
     2. モンスターが倒れていたらループを抜ける
     3. モンスターが勇者を攻撃する
  3. 倒れたほうの「たおれた！」を表示する
- **なぜ分けるのか**：`start()` の中に `for` と `while` を両方書くと、入れ子が深くなって読みづらくなります。
  **「敵を回す責任（`start`）」と「1体と戦う責任（`fight`）」を別のメソッドに分ける** と、どちらも短く保てます。
- **`private` にする理由**：外から「この1体とだけ戦って」と呼ばれるものではないから。公開する入口は `start()` だけのままです。

#### `private void showResult()`
- **やること**：勇者が生き残っていれば「全員撃破！」、倒れていれば「ゲームオーバー」を表示する。
- **ヒント**：判定は `hero.isAlive()` ひとつで足ります。モンスター側を調べる必要はありません（勇者が生きたままループを抜けた＝全部倒したから）。

## クラス仕様：`Main`

やることは変わりません。**用意するものが増えるだけ** です。

1. `Hero` を1体作る。
2. 空の `List<Monster>` を作り、モンスターを3体 `add` する（例：スライム / ゴブリン / ドラゴン）。
   - **強さは尻上がりにする**と連戦らしくなります（後ろほど HP・攻撃力を高く）。
3. 勇者とリストを渡して `Battle` を作り、`start()` を呼ぶ。

> ここで気づいてほしいこと：**敵を増やしたり順番を変えたりするのに、`Battle` も `Monster` も触る必要がない**。`Main` の `add` を足すだけで済みます。これが「役割を分けておくご褒美」です。

## 実装の進め方（おすすめ手順）

1. `Battle` の `start()` を、まるごと `fight(Monster monster)` にリネームする（引数でモンスターを受け取る形にする）。この時点では `Main` が動かなくなるので、次へ。
2. `Battle` のフィールドを `Monster monster` → `List<Monster> monsters` に変え、コンストラクタも合わせて直す。
3. 新しい `start()` を書く。中身は「拡張for文で `fight(m)` を呼ぶ」だけ。
4. `Main` でリストを作り、**まずモンスター1体だけ** `add` して実行する。フェーズ4と同じ出力になれば、ここまで成功。
5. モンスターを2体、3体と増やして実行する。連戦になることを確認する。
6. `start()` に「勇者が倒れていたら抜ける」判定を足す。わざと弱い勇者（HP 10 など）で試して、**ゲームオーバーで打ち切られる**ことを確認する。
7. `showResult()` を「全員撃破 / ゲームオーバー」の2パターンに直す。

```bash
javac -d out src/rpg/**/*.java src/rpg/*.java
java -cp out rpg.Main
```

> **つまずきポイント**：勇者のHPは戦いをまたいで **回復しません**。次の敵には削られた状態で挑みます。「思ったより勝てない」と感じたら、それはバグではなく仕様です（勇者のHPを上げるか、敵を弱くして調整してください）。
