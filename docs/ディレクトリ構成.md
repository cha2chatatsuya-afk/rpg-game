# ディレクトリ構成

「どこに何を置くか」で迷ったら、これを見てください。
ファイルは増減するので、各フォルダの役割だけまとめます。

## 見取り図

```
rpg-game/
├── .gitignore   … Git に登録しないファイルの設定
├── docs/        … 資料（この文書や要件定義書）
├── src/         … プログラム本体（.java）
│   └── rpg/     … このゲームのプログラム一式
│       └── model/ … 勇者・スライムなど「登場するもの」
└── out/         … コンパイル結果（.class）※自動生成・さわらない
```

補足：
- **パッケージ** = 関係するプログラムをまとめるフォルダ（`src/rpg`, `src/rpg/model`）。本棚をジャンルで分けるイメージ。
- **コンパイル** = `.java` を実行できる形（`.class`）に変換すること。

## 動かし方

`rpg-game/` の中で実行します。

```bash
# コンパイル（結果は out/ へ）
javac -d out src/rpg/**/*.java src/rpg/*.java

# 実行
java -cp out rpg.Main
```

## 追加するときの考え方

- 新しいプログラムは、似た仲間がいるフォルダに入れる（例：新キャラは `rpg/model`）。
- 別の役割（例：戦いの進行）が増えたら、新しいフォルダに分ける。
- 分け方を変えたら、この文書も直す。
